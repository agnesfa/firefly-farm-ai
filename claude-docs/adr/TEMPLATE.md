# NNNN — Short decision title

- **Status:** proposed | accepted | superseded by ADR NNNN | deprecated
- **Date:** YYYY-MM-DD
- **Authors:** Agnes, Claude, ...
- **Supersedes:** (list ADRs this replaces, if any)
- **Related:** (list related ADRs)

## Context

What's the situation that forced the decision? What constraints
exist? What changed (or broke) that made this decision necessary?
Keep this factual — no hindsight narrative. If there was an
incident, link the session summary or team memory entry.

## Decision

What we decided. One paragraph. Prefer concrete nouns and verbs
over adjectives. If the decision has multiple parts, list them.

## Rationale

Why this and not the alternatives? What trade-offs did we accept?
This is the most important section — it's what makes the ADR
worth reading years later.

### Alternatives considered

- **Option A:** brief description — why rejected
- **Option B:** brief description — why rejected

## Consequences

### Positive

- What this unlocks
- What problems it eliminates
- What it makes simpler

### Negative

- What this costs
- What it constrains going forward
- What risks we accept

### Neutral

- What changes in behaviour that isn't strictly better or worse

## Implementation

Pointer to the code. Not a reproduction of the code. Include:

- Files changed (main paths)
- Commit SHA when this ADR was written
- Any follow-up tasks that are explicitly deferred

## Open questions

Anything we punted. Each question names the person who owns it.
